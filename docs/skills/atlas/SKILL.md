# skills/atlas/SKILL.md

The `atlas` skill file. Defines activation and minimum rules for the main agent.

## Contents

- `description` in frontmatter (English, used by Claude Code for automatic activation) — activates in repos with `AGENTS.md` + `docs/` at the root.
- Instruction to the agent: the first action in a session is to read the repo's `AGENTS.md`; methodology and per-repo mapping live there.
- 4 fallback rules: update the matching doc in the same turn; no matching doc → create one; no aspirational content; stale content — delete. Plus rule 5 — closing ritual: run `/atlas:sync --check` before declaring a task done.
- Command list: `/atlas:init`, `/atlas:sync`, `/atlas:sync --check`.
- Short description of `mirrors:`-frontmatter (how parsers — the hook and auditor — resolve the mapping).
- One-line summary format for the end of code-touching turns.

## Size

~60 lines. The skill is a minimal runner. The extended methodology lives in the repo's own AGENTS.md (generated by `/atlas:init`).
